# AGENTS.md - Coding Agent Guidelines for ReichbR

This document provides guidelines for AI coding agents working in the ReichbR R package.

## Project Overview

**ReichbR** is an R package providing tools for SMC (Science Media Center) hospital
accessibility analyses ("Krankenhaus Erreichbarkeitsanalysen"). It calculates driving
times to hospitals in Germany using PostgreSQL/DuckDB databases and geospatial data.

- **Language**: R (>= 4.5.0)
- **Package Type**: Standard R package using devtools/roxygen2
- **Testing Framework**: testthat (edition 3)
- **Domain**: Geospatial analysis, database operations, data transformation

## Build, Test, and Lint Commands

### Primary Development Commands (via devtools)

```r
# Document the package (regenerate NAMESPACE and man/ files)
devtools::document()

# Load all package functions for interactive testing
devtools::load_all()

devtools::test()
```

### Running a Single Test

```r

testthat::test_file("tests/testthat/test-<name>.R")

devtools::test_active_file()

# Run tests matching a pattern
devtools::test(filter = "pattern")
```


## Code Style Guidelines

### Naming Conventions

| Element              | Convention                | Examples                                    |
|---------------------|---------------------------|---------------------------------------------|
| Internal functions  | `snake_case`              | `connect_to_postgres_db`, `ensure_dir_exists` |
| Public/user functions| German `PascalCase`      | `Fahrzeit_Zusammenfassung`, `Geofilter_Funktion` |
| Parameters          | `snake_case`              | `pg_schema`, `filter_column_name`           |
| Column names        | German `PascalCase`       | `Gemeindeschluessel`, `Gitterzellen_ID`     |
| Local variables     | `snake_case`              | `result`, `con_duck`, `table_counts`        |

### Import Management

All imports are declared in `R/zzz.R` using roxygen2 `@importFrom` tags:

```r
#' @importFrom dplyr mutate left_join summarise filter distinct
#' @importFrom DBI dbConnect dbExecute dbGetQuery dbDisconnect
#' @importFrom glue glue glue_sql
```

**Do NOT use `library()` in package code.** Use explicit imports via `@importFrom`.

### Global Variables

Declare column names used in NSE (non-standard evaluation) at the top of each file:

```r
utils::globalVariables(c(
  "Gemeindeschluessel",
  "Gitterzellen_ID",
  "Fahrzeit_Sekunden"
))
```

### Documentation (roxygen2)

All exported functions must have roxygen2 documentation:

```r
#' Short Title (One Line)
#'
#' Longer description of what the function does.
#' All tabular inputs and outputs are expected to be tibbles.
#'
#' @param data A tibble with required columns.
#' @param .by Grouping columns (tidyselect).
#' @param threshold Optional numeric threshold.
#'
#' @return A tibble with results.
#'
#' @examples
#' \dontrun{
#' result <- my_function(df, threshold = 30)
#' }
#'
#' @export
```

### Pipe Style

Use the native R pipe `|>` (not magrittr `%>%`):

```r
result <- data |>
  mutate(new_col = old_col / 60) |>
  filter(value > threshold) |>
  summarise(total = sum(count), .by = group_col)
```

### SQL Query Construction

Use `glue::glue_sql()` for safe SQL query construction:

```r
sql <- glue_sql(
  "SELECT * FROM {`pg_schema`}.{`table_name`} WHERE id = {id}",
  .con = con
)
dbExecute(con, sql)
```

### Error Handling

```r
# Database connections with tryCatch
con <- tryCatch(
  dbConnect(Postgres(), ...),
  error = function(e) stop("Could not connect: ", e$message)
)

# File existence checks
require_file_exists(file_path)  # Stops if file doesn't exist
ensure_dir_exists(dir_path)      # Creates directory if missing

# Type validation
if (!tibble::is_tibble(data)) {
  stop("Expected a tibble, got ", class(data)[1])
}

# Required columns check
missing_cols <- setdiff(required_cols, names(data))
if (length(missing_cols) > 0) {
  stop("Missing required columns: ", paste(missing_cols, collapse = ", "))
}
```

### Function Return Values

- Functions should return tibbles consistently
- Use `tibble::as_tibble()` to ensure return type
- Use `invisible()` for side-effect functions (e.g., `dbExecute` wrappers)

```r
# Return tibble explicitly
tibble::as_tibble(result)

# Side-effect function
invisible(dbExecute(con, sql))
```

### Code Section Dividers

Use comment dividers for major code sections:

```r

# Section Name
# -------------------------------------------------------------------- #
```

## Testing Guidelines

Test files should be named `test-<feature>.R` in `tests/testthat/`:

```r
test_that("function_name handles edge case", {
  result <- function_name(input)
  expect_true(tibble::is_tibble(result))
  expect_equal(nrow(result), expected_rows)
})
```

## Agent Permissions

### Allowed Without Permission

Read-only operations that can be executed freely:

- **Shell**: `find`, `grep`, `rg`, `cat`, `head`, `tail`, `ls`, `tree`
- **Git inspection**: `git log`, `git status`, `git diff`, `git branch`, `git show`
- **Tools**: `Read`, `Glob`, `Grep`, `Task` (explore)

### Requires Explicit Permission

Operations that modify state - ask user before executing:

- **R package commands**: `devtools::document()`, `devtools::test()`, `devtools::check()`
- **File modifications**: `Edit` tool, `Write` tool
- **Build commands**: `R CMD build`, `R CMD INSTALL`
- **Creating temporary files** for testing or debugging

### Forbidden

Never execute under any circumstances:

- **Destructive git**: `git push --force`, `git reset --hard`, `git clean -fd`
- **Git write ops**: `git commit`, `git push`, `git merge`, `git rebase`
- **Recursive delete**: `rm -rf`, `rm -r`
- **System-level**: `sudo`, `chmod`, `chown`, `systemctl`
- **Credentials**: Reading `.env`, `credentials.json`, SSH keys, API tokens
- **Process control**: `kill`, `pkill`, `killall`

### Permission Request Protocol

When requesting permission, state:
1. The exact command or operation
2. The purpose and expected outcome
3. Any potential side effects

On denial or no response: abort and suggest alternatives.
